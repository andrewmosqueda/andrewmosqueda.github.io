CURRENT MODE
CONVERTER

(LTSpice simulation)

The conventional buck converter with the PID controller generates the PWM signal based by comparing the sum of the outputs of the PID to a fix sawtooth signal. The PWM output initially turns HIGH every start of a single switching cycle and then turns LOW when the sawtooth signal reaches the PID output value. In the current mode converter shown on the right, the fix sawtooth signal is replaced by a signal sensed from the inductor current. The element "V2" on the schematic on the right is the inductor current sensor and its output is fed into the CFEED of the controller. The LTSpice simulation files I have created for the current mode converter can be downloaded here.

The schematic of the current mode controller is shown below. The controller is just a modified PID controller in which the current of the inductor (CFEED input) is used as the comparator signal instead of the fix sawtooth signal. I also made the clock signal (SAW input) as external so that the switching frequency can be changed in your simulation. I have also made POUT, IOUT, DOUT, SET, RESET and PID_SH_CLK as debug outputs so it can be monitored if needed during simulation. As opposed to a fix sawtooth compared signal, the inductor current sensed signal will decrease once the PWM signal turns LOW so an SR flip-flop system is used so to maintain the LOW PWM signal after the inductor current reaches the PID value. The top trace of the Draft2.raw window below represents the load of the 5V converter. To simulate real world scenario, the load jumps to 4A at time=0.3ms and then return to low load at time=1.3ms. The middle trace compares the output of the PID (POUT+IOUT+DOUT) and the inductor current. The bottom trace is the 5V output.
